# Data Quality Report

## Overview

This report summarizes the data quality assessment for the churn dataset and related sources.

## Data sources

- `data/customers.csv`
- `data/orders.csv`
- `data/web_events_snapshot.csv`
- `data/support_tickets.csv`
- `data/intervention_history.csv`
- `data/churn_labels.csv`

## Key findings

- Missing value patterns and affected fields
- Data consistency issues across datasets
- Duplicate records and invalid entries
- Anomalies in date and numeric fields

## Recommended actions

- Clean missing values and validate data types
- Resolve duplicate or inconsistent customer records
- Reconcile mismatched timestamps or event records
- Use data profiling and monitoring for future dataset updates
